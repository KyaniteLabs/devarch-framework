# Report Structure

Sections and organization for ARCHAEOLOGY-REPORT files.

## Section Order

### 1. Executive Summary

High-level overview of findings.

- Project name and description
- Total commits analyzed
- Time period covered
- Key findings (3-5 bullet points)
- Overall assessment

### 2. Project Overview

Context about the analyzed project.

- Developer name and handle
- Repository path
- Project description
- Analysis date
- Configuration summary

### 3. Data Collection Summary

How data was extracted and processed.

- Repository extraction method
- Commit count and date range
- Authors identified
- Files touched
- Data quality notes

### 4. Signal Detection Results

Summary of detected signals.

- Total signals detected
- Breakdown by type
- Notable signals (top 5-10)
- Signal distribution over time

### 5. Analysis Findings

Results from analysis vectors.

- SDLC gap findings
- ML pattern findings
- Formal terms findings
- Source archaeology findings
- Each vector with key insights

### 6. Supplementary Correlations

External data correlations (if configured).

- Sources integrated
- Key correlations found
- Notable patterns
- Insights from cross-referencing

### 7. Visualizations

References to charts and graphs.

- Timeline of activity
- Velocity shifts
- Author distribution
- Correlation plots
- Activity heatmap

### 8. Recommendations

Actionable insights based on analysis.

- Areas for improvement
- Patterns to continue
- Risks identified
- Optimization opportunities

### 9. Appendix

Supporting information.

- Methodology details
- Configuration details
- Supplementary data sources
- Glossary of terms

## Format Guidelines

- Use markdown headers (# ## ###) for structure
- Include bullet points for lists
- Use tables for numerical summaries
- Link to visualizations where relevant
- Keep sections under 200 lines each

## HTML Version

HTML report includes same sections with:

- Table of contents with anchor links
- Embedded charts
- Expandable sections
- Print-friendly layout
